🎯 SOLUTION RAPIDE (COPIER-COLLER)
═══════════════════════════════════════════════════════════════════════════

Pour projets > 50 MB:

git config --global core.compression 0
git config --global http.postBuffer 1048576000
git config --global http.lowSpeedTime 999999
git config --global http.lowSpeedLimit 0
git push origin master --verbose


📊 RÉSUMÉ DU PROBLÈME
═══════════════════════════════════════════════════════════════════════════

Projet: ClaraVerse V5
Taille: 75+ MB
Problème: Timeout lors du push vers GitHub
Solution: Désactivation compression + augmentation buffers
Résultat: ✅ Push réussi


💡 POINTS CLÉS
═══════════════════════════════════════════════════════════════════════════

• Les paramètres Git par défaut ne conviennent pas aux gros projets
• La compression peut être contre-productive pour gros transferts
• SSH est plus stable que HTTPS pour gros projets
• GitHub Desktop est la solution la plus simple
• Plusieurs tentatives peuvent être nécessaires
